pub fn play_audio_based_on_event( &mut self, event: &Event ) -> Result<(), OxAgAudioToolError>

Processes an event and plays the appropriate audio for it

If it’s a Event::Moved it will detect whether the TileType on which the robot is on changed and if it did it plays the configured sound for it, stopping the one playing previously for the different TileType.

If it’s either a Event::TimeChanged or a Event::DayChanged it will detect whether the WeatherType has changed and if it did it plays the configured sound for it, stopping the one playing previously for the different WeatherType.

If it’s any other event, it checks whether a sound is configured for it and plays it
